Quality of Service

In this page, you can create a traffic class rule to classify the upstream traffic, assign queue that defines the precedence and the interface, and optionally overwrite the DSCP byte of the IP header. A rule consists of a class name and at least one condition. All the specified conditions in the class rule take effect simultaneously. Click "Apply" to save the rule and enable it.

Assign ATM Priority and/or DSCP Mark for the class
If a non-blank value is set for "Assign Differentiated Services Code Point (DSCP) Mark", the corresponding DSCP byte in the IP header of the upstream packet is overwritten by the set value.

Helpful Hints...

Quality of service (QoS) enables the router to set priority of data packets that flow throughout your network. It is a useful feature for sensitive applications such as VoIP whereby it assists in preventing dropped calls. Large amount of non-critical data can be scaled, to avoid affecting sensitive real-time applications such as VoIP or streaming. This page shows some pre-configured rules that are used with common applications such as VoIP, FTP, and messaging programs.
